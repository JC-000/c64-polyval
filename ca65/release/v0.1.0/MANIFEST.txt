Notes
-----
- polyval_long.lib and polyval_short.lib are mutually exclusive. A consumer
  picks exactly one and links against it.
- The ar65 archiver embeds a small amount of build-time metadata per .o
  module (~12 bytes total across the archive), so re-running `make release`
  produces archives of identical size and identical object-module contents
  but not a byte-identical .lib file. MD5s above are for the specific build
  recorded by this manifest.
- abi_v1.inc is the stable consumer-facing ABI. test_probes.inc is NOT
  stable and exists only so the upstream test suite can reach inner
  helpers. See README.md for details.

Next steps (user-only - NOT done by this release build)

  and /home/someone/c64-polyval/tools/ once downstream consumers have
  migrated off it. This release only adds files under ca65/release/ and
  a new Makefile target; nothing in src/ or tools/ has been touched.
